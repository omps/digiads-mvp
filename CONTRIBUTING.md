# Contributing to DigiAds MVP

Thank you for your interest in contributing to DigiAds MVP! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/digiads-mvp.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit with clear messages: `git commit -m "Add feature: description"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Code Style

- Use ES6+ features
- Follow existing code formatting
- Add comments for complex logic
- Write meaningful variable and function names
- Use async/await instead of callbacks

## Testing

- Write unit tests for new features
- Ensure all tests pass before submitting PR
- Test all microservices integration

## Pull Request Process

1. Update README.md with any new features or changes
2. Update API documentation if endpoints change
3. Ensure your code follows the existing style
4. Link any related issues in the PR description
5. Request review from maintainers

## Reporting Bugs

When reporting bugs, please include:

- Node.js version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages/logs

## Feature Requests

We welcome feature requests! Please:

- Check existing issues first
- Clearly describe the feature
- Explain the use case
- Provide examples if possible

## Code of Conduct

- Be respectful and inclusive
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy towards others
